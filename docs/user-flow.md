# 사용자 플로우(User Flow)

## 📱 “Catalog” 앱 사용자 플로우

### 1️⃣ 시작 및 로그인

* **Splash Screen 1 → Splash Screen 2**

  * 앱 로고와 시그니처 컬러가 애니메이션 형태로 표시됨.
  * 초기 로딩 후 로그인 화면으로 전환.
* **로그인 화면**

  * SNS 계정(Naver, Google, Apple 등) 로그인 방식.
  * 첫 로그인 시 사용자 정보 자동 수집 → 프로필 생성.

---

### 2️⃣ 프로필 설정

* **화면:** `프로필 편집`
* **주요 기능:**

  * 프로필 이미지 업로드
  * 닉네임/이메일 수정
  * 회원 탈퇴 기능 포함
* **저장 시 이동:** → `홈` 화면으로 진입

---

### 3️⃣ 홈 탭 (전체 카탈로그 요약)

* **화면:** `홈`
* **내용:**

  * 사용자의 전체 카탈로그 목록 표시
  * 각 카탈로그별 진행률(보유/전체 아이템 수) 시각화
* **행동:**

  * 특정 카탈로그 클릭 → `카탈로그 상세`
  * 하단 네비게이션을 통해 탐색(탐색/추가/마이) 가능

---

### 4️⃣ 탐색 탭 (검색 및 추천)

* **화면:** `카탈로그 탐색`
* **기능 구조:**

  * 상단 검색창 (예: “스니커즈”, “TCG”)
  * 필터: [인기 카탈로그] / [신규 카탈로그]
  * 검색 결과: 카드 리스트 (썸네일 + 제목 + 설명 + 아이템 수)
* **행동:**

  * 결과 항목 클릭 → `카탈로그 상세(저장 전)`
  * “+ 내 카탈로그에 저장” → `카탈로그 상세(저장 후)` 상태로 전환

---

### 5️⃣ 카탈로그 상세

* **2가지 상태:**

  * **저장 전:** “+ 내 카탈로그에 저장” 버튼 노출
  * **저장 후:** `편집하기 / 삭제하기` 기능 노출
* **구성:**

  * 카탈로그 썸네일, 설명, 총 아이템 수, 보유 현황
  * 아이템 카드 리스트(보유/미보유/가격 표시)
* **행동:**

  * 아이템 클릭 → `아이템 상세`
  * `편집하기` 클릭 → 카탈로그 정보 수정
  * `삭제하기` 클릭 → 카탈로그 제거

---

### 6️⃣ 아이템 상세

* **화면:** `아이템 상세`
* **내용:**

  * 이미지 슬라이드, 기본 정보(시리즈, 제작년도, 디자이너, 가격 등)
  * 인증 링크(Certificate of Authenticity)
* **행동:**

  * “수집 완료하기” 클릭 시 → 상태 변경 및 애니메이션 효과
  * “수집 완료!” 버튼으로 변경, 별·하트 등 시각적 피드백 표시

---

### 7️⃣ 추가 탭

* **화면:** `추가`
* **기능:**

  * “새 카탈로그 추가”
  * “기존 카탈로그에 새 아이템 추가”
* **각 선택 시:**

  * 새 카탈로그 추가 → `카탈로그 생성 폼`
  * 기존 카탈로그 선택 → `새 아이템 추가` 단계로 연결

---

### 8️⃣ 마이 탭

* **화면:** `마이` → `프로필 편집`
* **내용:**

  * 프로필 이미지, 닉네임, 이메일 확인
  * 설정/로그아웃/탈퇴 기능

---

## 🧩 전체 구조 요약

```plaintext
[스플래시]
   ↓
[로그인]
   ↓
[홈] ⇄ [탐색] ⇄ [추가] ⇄ [마이]
   ↓             ↓
[카탈로그 상세]  [새 카탈로그 / 아이템 추가]
   ↓
[아이템 상세]
   ↓
[수집 완료 애니메이션]
```

---

## 💡 UX 관점에서의 특징

| 구분            | 설명                                      |
| ------------- | --------------------------------------- |
| **핵심 사용자 흐름** | 로그인 → 홈 → 카탈로그 상세 → 아이템 상세 → 수집 완료      |
| **보조 흐름**     | 탐색을 통한 타인 카탈로그 확인 및 저장, 새 카탈로그 생성       |
| **UI 일관성**    | 상단 네비게이션(뒤로가기 + 홈)과 하단 탭 4분할 구조 유지      |
| **피드백 디자인**   | 수집 완료 시 즉각적 시각 효과 제공 (별, 하트, 파티클 애니메이션) |
| **데이터 중심성**   | 카탈로그 단위의 진척도 시각화로 수집 성취감 강화             |
