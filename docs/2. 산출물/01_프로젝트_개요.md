# 프로젝트 개요 (Project Overview)

## 1. 목적

카탈로깅(Cataloging)은 수집가들이 자신의 소장품을 체계적으로 기록하고 관리할 수 있는 디지털 플랫폼입니다. 넘버링 시리즈, 피규어, 음반, 스니커즈 등 다양한 수집품의 보유 현황을 한눈에 파악하고, 수집률을 실시간으로 확인하여 완성의 성취감을 제공합니다.

### 핵심 목표

- **체계적인 수집 관리**: 카탈로그 단위로 수집품을 분류하고 관리
- **보유 현황 추적**: 보유/미보유 아이템을 명확히 구분하여 누락 방지
- **수집률 시각화**: 실시간 수집률 계산으로 완성도 확인
- **데이터 안전성**: JWT 기반 인증으로 사용자별 데이터 완전 격리

## 2. 주요 기능

### 2.1 사용자 관리 (User API - Spring Boot)

- **SNS 로그인**: Google, Naver OAuth2 기반 소셜 로그인
- **개발용 간편 로그인**: 이메일/닉네임만으로 빠른 테스트 가능
- **JWT 토큰 발급**: HS256 알고리즘, 24시간 유효기간
- **프로필 관리**: 닉네임, 자기소개, 프로필 이미지 수정
- **회원 탈퇴**: Soft Delete 방식으로 데이터 보존

### 2.2 카탈로그 관리 (Catalog API - FastAPI)

- **카탈로그 CRUD**: 생성, 조회, 수정, 삭제
- **카테고리 분류**: 피규어, 레고, 음반 등 자유로운 분류
- **태그 시스템**: 다중 태그로 세밀한 분류 가능
- **공개/비공개 설정**: 카탈로그 공개 범위 제어
- **카탈로그 저장**: 다른 사용자의 공개 카탈로그를 내 컬렉션에 복사

### 2.3 아이템 관리 (Catalog API - FastAPI)

- **아이템 CRUD**: 생성, 조회, 수정, 삭제
- **보유 상태 토글**: 클릭 한 번으로 보유/미보유 전환
- **사용자 정의 필드**: JSON 형태로 자유로운 메타데이터 저장
- **이미지 업로드**: 아이템별 이미지 첨부 가능
- **실시간 수집률**: 보유 아이템 수 / 전체 아이템 수 × 100

### 2.4 클라이언트 기능 (Flutter)

- **크로스 플랫폼**: 웹, iOS, Android, macOS 지원
- **반응형 UI**: GetX 상태 관리로 실시간 UI 업데이트
- **자동 로그인**: JWT 토큰 저장으로 재로그인 불필요
- **오프라인 대응**: 네트워크 오류 시 적절한 안내 메시지
- **플랫폼별 최적화**: 웹/모바일 환경에 맞는 API URL 자동 설정

## 3. 기대 효과

### 3.1 사용자 관점

- **중복 구매 방지**: 보유 현황을 즉시 확인하여 불필요한 구매 방지
- **누락 아이템 파악**: 미보유 필터로 빠진 아이템 신속 확인
- **수집 성취감**: 수집률 퍼센티지로 완성도 시각화
- **체계적 관리**: 카테고리/태그로 대량의 수집품 효율적 정리

### 3.2 기술적 효과

- **마이크로서비스 아키텍처**: 기능별 서비스 분리로 확장성 확보
- **JWT 기반 인증**: Stateless 인증으로 서버 부하 최소화
- **데이터 격리**: 사용자별 완전한 데이터 분리로 보안 강화
- **실시간 동기화**: 클라이언트-서버 간 즉각적인 데이터 반영

### 3.3 비즈니스 효과

- **사용자 확장성**: 마이크로서비스로 트래픽 증가에 유연 대응
- **개발 생산성**: 명확한 API 분리로 병렬 개발 가능
- **유지보수성**: 레이어별 책임 분리로 코드 관리 용이
- **향후 확장**: 커뮤니티, 거래, OCR 등 추가 기능 통합 용이

## 4. 개발 현황

### 4.1 개발 기간

- **시작일**: 2025년 9월 5일
- **예정 완료일**: 2025년 12월 6일
- **현재 상태**: MVP 개발 완료, 로컬 환경 테스트 중

### 4.2 개발 환경

- **운영체제**: macOS
- **프론트엔드**: Flutter 3.x
- **백엔드**: Spring Boot 3.2.0 (Java 17), FastAPI (Python 3.13)
- **데이터베이스**: H2 (개발용), SQLite (개발용)
- **가상환경**: Python venv (Catalog API - `be/catalog-api/.venv`)
- **버전 관리**: Git

### 4.3 배포 환경

**현재 (로컬 개발)**
- Flutter 클라이언트: http://localhost:3000
- Spring Boot User API: http://localhost:8080 (기본값, PORT 환경 변수로 변경 가능)
- FastAPI Catalog API: http://localhost:8000 (기본값, .env 파일의 PORT로 변경 가능)

**향후 (프로덕션)**
- 클라이언트: AWS S3 + CloudFront
- API 서버: AWS EC2 + ALB
- 데이터베이스: AWS RDS (User API), DynamoDB (Catalog API)
- 이미지 저장: AWS S3

## 5. 프로젝트 특징

### 5.1 마이크로서비스 아키텍처

- **User API**: 회원 관리 및 JWT 토큰 발급 전담
- **Catalog API**: 카탈로그/아이템 관리 전담
- **독립적 확장**: 각 서비스를 독립적으로 배포 및 확장 가능
- **기술 스택 다양화**: Spring Boot와 FastAPI의 장점 활용

### 5.2 JWT 기반 통합 인증

- **단일 시크릿 키**: 두 서비스가 동일한 JWT 시크릿 공유
- **토큰 발급**: User API에서 로그인 시 JWT 발급
- **토큰 검증**: Catalog API에서 동일한 키로 검증
- **사용자 식별**: JWT의 sub 클레임에서 user_id 추출

### 5.3 실시간 수집률 계산

- **동적 계산**: API 호출 시마다 최신 수집률 계산
- **사용자별 상태**: user_item_status 테이블로 다중 사용자 지원
- **즉각 반영**: 보유 상태 변경 시 UI에 즉시 반영

### 5.4 완전한 통신 로깅

- **요청/응답 로깅**: 모든 API 통신 내역 자동 기록
- **성능 측정**: 응답 시간, 데이터 크기 추적
- **디버깅 용이**: 상세한 로그로 문제 진단 간소화
