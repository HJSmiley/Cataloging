# UI/UX 문서

## 1. 화면 목록

카탈로깅 앱은 총 11개의 화면으로 구성됩니다.

| 번호 | 화면명 | 파일명 | 설명 |
|------|--------|--------|------|
| 1 | 스플래시 | splash_screen.dart | 앱 시작 화면, 자동 로그인 처리 |
| 2 | 로그인 | login_screen.dart | 개발용 로그인 (이메일/닉네임 입력) |
| 3 | 홈 | home_screen.dart | 내 카탈로그 목록 및 수집률 표시 |
| 4 | 탐색 | explore_screen.dart | 공개 카탈로그 탐색 |
| 5 | 추가 | add_screen.dart | 카탈로그 또는 아이템 추가 선택 |
| 6 | 마이 | my_screen.dart | 프로필 및 설정 |
| 7 | 카탈로그 상세 | catalog_detail_screen.dart | 카탈로그 정보 및 아이템 목록 |
| 8 | 카탈로그 편집 | catalog_edit_screen.dart | 카탈로그 생성/수정 |
| 9 | 아이템 추가 | item_add_screen.dart | 새 아이템 추가 |
| 10 | 아이템 상세 | item_detail_screen.dart | 아이템 정보 및 보유 상태 |
| 11 | 프로필 편집 | profile_edit_screen.dart | 사용자 프로필 수정 |

## 2. 화면 간 이동 흐름

### 2.1 전체 흐름도

```
[SplashScreen]
     │
     ├─ 토큰 있음 → [HomeScreen]
     └─ 토큰 없음 → [LoginScreen] → [HomeScreen]
                                          │
                    ┌─────────────────────┼─────────────────────┐
                    │                     │                     │
              [ExploreScreen]        [AddScreen]          [MyScreen]
                    │                     │                     │
                    │              ┌──────┴──────┐              │
                    │              │             │              │
                    │    [CatalogEditScreen] [ItemAddScreen]   │
                    │                                           │
                    │                                    [ProfileEditScreen]
                    │
            [CatalogDetailScreen]
                    │
                    ├─ [CatalogEditScreen]
                    └─ [ItemDetailScreen]
```

### 2.2 주요 네비게이션

**하단 탭 네비게이션** (4개):
1. 홈 (HomeScreen)
2. 탐색 (ExploreScreen)
3. 추가 (AddScreen)
4. 마이 (MyScreen)

**스택 네비게이션**:
- 카탈로그 상세 → 아이템 상세
- 카탈로그 상세 → 카탈로그 편집
- 추가 → 카탈로그 편집 / 아이템 추가
- 마이 → 프로필 편집

## 3. 화면별 상세 설명

### 3.1 SplashScreen (스플래시)

**목적**: 앱 로딩 및 자동 로그인 처리

**주요 기능**:
- 앱 로고 애니메이션 (페이드 인 + 스케일)
- SharedPreferences에서 JWT 토큰 로드
- 토큰 유효성 검증
- 3초 후 자동 전환

**화면 전환**:
- 토큰 유효 → HomeScreen
- 토큰 없음/만료 → LoginScreen

### 3.2 LoginScreen (로그인)

**목적**: 사용자 인증

**주요 기능**:
- 이메일 입력 필드
- 닉네임 입력 필드
- 로그인 버튼
- 개발용 간편 로그인 (OAuth2는 향후 추가)

**입력 검증**:
- 이메일 형식 검증
- 닉네임 2자 이상

**화면 전환**:
- 로그인 성공 → HomeScreen

### 3.3 HomeScreen (홈)

**목적**: 내 카탈로그 목록 표시

**주요 기능**:
- 카탈로그 카드 리스트
- 각 카드에 제목, 설명, 수집률 표시
- 수집률 프로그레스 바
- 새로고침 (Pull to Refresh)

**카탈로그 카드 정보**:
- 썸네일 이미지
- 제목
- 카테고리
- 아이템 수 (보유/전체)
- 수집률 (%)

**화면 전환**:
- 카드 클릭 → CatalogDetailScreen

### 3.4 ExploreScreen (탐색)

**목적**: 공개 카탈로그 탐색

**주요 기능**:
- 공개 카탈로그 목록
- 카테고리 필터
- 검색 기능 (향후 추가)

**화면 전환**:
- 카드 클릭 → CatalogDetailScreen (저장 버튼 표시)

### 3.5 AddScreen (추가)

**목적**: 카탈로그 또는 아이템 추가 선택

**주요 기능**:
- "새 카탈로그 추가" 버튼
- "기존 카탈로그에 아이템 추가" 버튼

**화면 전환**:
- 새 카탈로그 → CatalogEditScreen
- 아이템 추가 → 카탈로그 선택 → ItemAddScreen

### 3.6 MyScreen (마이)

**목적**: 프로필 및 설정

**주요 기능**:
- 프로필 이미지
- 닉네임
- 이메일
- 프로필 편집 버튼
- 로그아웃 버튼

**화면 전환**:
- 프로필 편집 → ProfileEditScreen

### 3.7 CatalogDetailScreen (카탈로그 상세)

**목적**: 카탈로그 정보 및 아이템 목록 표시

**주요 기능**:
- 카탈로그 정보 (제목, 설명, 카테고리, 태그)
- 수집률 표시 (큰 프로그레스 바)
- 아이템 목록 (그리드 또는 리스트)
- 아이템 보유 상태 체크박스
- 필터 (전체/보유/미보유)
- 편집/삭제 버튼 (소유자인 경우)
- 저장 버튼 (공개 카탈로그인 경우)

**아이템 카드 정보**:
- 이미지
- 이름
- 보유 여부 체크박스

**화면 전환**:
- 아이템 클릭 → ItemDetailScreen
- 편집 버튼 → CatalogEditScreen
- 아이템 추가 버튼 → ItemAddScreen

### 3.8 CatalogEditScreen (카탈로그 편집)

**목적**: 카탈로그 생성 또는 수정

**주요 기능**:
- 제목 입력 필드 (필수)
- 설명 입력 필드 (필수)
- 카테고리 선택
- 태그 입력 (쉼표로 구분)
- 공개 여부 토글
- 썸네일 이미지 업로드
- 저장 버튼

**입력 검증**:
- 제목 1자 이상
- 설명 1자 이상

**화면 전환**:
- 저장 성공 → 이전 화면으로 돌아가기

### 3.9 ItemAddScreen (아이템 추가)

**목적**: 새 아이템 추가

**주요 기능**:
- 이름 입력 필드 (필수)
- 설명 입력 필드 (필수)
- 이미지 업로드
- 사용자 정의 필드 추가 (키-값 쌍)
- 저장 버튼

**사용자 정의 필드 예시**:
- 제조사, 시리즈, 출시일, 가격 등

**화면 전환**:
- 저장 성공 → CatalogDetailScreen

### 3.10 ItemDetailScreen (아이템 상세)

**목적**: 아이템 정보 표시 및 보유 상태 관리

**주요 기능**:
- 이미지 (큰 크기)
- 이름
- 설명
- 사용자 정의 필드 표시
- 보유 상태 토글 버튼
- 수정/삭제 버튼 (소유자인 경우)

**화면 전환**:
- 수정 버튼 → 편집 모드
- 삭제 성공 → CatalogDetailScreen

### 3.11 ProfileEditScreen (프로필 편집)

**목적**: 사용자 프로필 수정

**주요 기능**:
- 프로필 이미지 업로드
- 닉네임 입력 필드
- 자기소개 입력 필드
- 저장 버튼
- 회원 탈퇴 버튼

**화면 전환**:
- 저장 성공 → MyScreen
- 회원 탈퇴 → LoginScreen

## 4. 사용자 시나리오

### 4.1 신규 사용자 시나리오

```
1. 앱 실행 (SplashScreen)
2. 로그인 (LoginScreen)
   - 이메일: user@example.com
   - 닉네임: 수집가
3. 홈 화면 (HomeScreen) - 빈 상태
4. 추가 화면 (AddScreen)
5. 새 카탈로그 추가 (CatalogEditScreen)
   - 제목: 내 피규어 컬렉션
   - 설명: 애니메이션 피규어 모음
   - 카테고리: 피규어
6. 카탈로그 상세 (CatalogDetailScreen) - 빈 상태
7. 아이템 추가 (ItemAddScreen)
   - 이름: 하츠네 미쿠 피규어
   - 설명: 1/8 스케일
8. 카탈로그 상세 (CatalogDetailScreen)
   - 아이템 1개 표시
   - 수집률: 0% (미보유)
9. 아이템 체크박스 클릭 → 보유 상태 토글
   - 수집률: 100% (보유)
```

### 4.2 기존 사용자 시나리오

```
1. 앱 실행 (SplashScreen)
2. 자동 로그인 → 홈 화면 (HomeScreen)
3. 카탈로그 목록 표시
   - 피규어 컬렉션 (70% 완성)
   - 레고 세트 (50% 완성)
4. 피규어 컬렉션 클릭 (CatalogDetailScreen)
5. 미보유 필터 적용
   - 미보유 아이템만 표시
6. 누락 아이템 확인 및 구매 계획 수립
```

### 4.3 카탈로그 저장 시나리오

```
1. 탐색 화면 (ExploreScreen)
2. 공개 카탈로그 목록 표시
3. 관심 있는 카탈로그 클릭 (CatalogDetailScreen)
4. "내 카탈로그에 저장" 버튼 클릭
5. 저장 성공 메시지
6. 홈 화면 (HomeScreen)으로 이동
7. 저장된 카탈로그 표시 (수집률 0%)
8. 카탈로그 상세 (CatalogDetailScreen)
9. 아이템 보유 상태 관리 시작
```

## 5. 디자인 가이드라인

### 5.1 색상 팔레트

- **Primary**: 앱의 주요 색상 (로고, 버튼)
- **Secondary**: 보조 색상 (강조, 링크)
- **Background**: 배경 색상 (밝은 회색)
- **Surface**: 카드 배경 (흰색)
- **Error**: 에러 메시지 (빨간색)
- **Success**: 성공 메시지 (초록색)

### 5.2 타이포그래피

- **제목**: 20-24px, Bold
- **부제목**: 16-18px, Medium
- **본문**: 14-16px, Regular
- **캡션**: 12-14px, Regular

### 5.3 간격 및 여백

- **화면 여백**: 16px
- **카드 간격**: 12px
- **요소 간격**: 8px
- **버튼 높이**: 48px

### 5.4 컴포넌트

**카탈로그 카드**:
- 썸네일 이미지 (정사각형)
- 제목 (1줄, 말줄임)
- 카테고리 태그
- 수집률 프로그레스 바
- 아이템 수 (보유/전체)

**아이템 카드**:
- 이미지 (정사각형)
- 이름 (1줄, 말줄임)
- 보유 상태 체크박스

**프로그레스 바**:
- 높이: 8px
- 둥근 모서리
- 색상: Primary (진행), 회색 (배경)
- 퍼센티지 텍스트 표시

## 6. 반응형 디자인

### 6.1 웹 (데스크톱)

- 최대 너비: 1200px
- 카탈로그 카드: 3열 그리드
- 아이템 카드: 4-5열 그리드

### 6.2 태블릿

- 카탈로그 카드: 2열 그리드
- 아이템 카드: 3-4열 그리드

### 6.3 모바일

- 카탈로그 카드: 1열 리스트
- 아이템 카드: 2열 그리드

## 7. 접근성

- **키보드 네비게이션**: Tab 키로 이동 가능
- **스크린 리더**: 모든 요소에 적절한 레이블
- **색상 대비**: WCAG AA 기준 준수
- **터치 타겟**: 최소 44x44px

## 8. 애니메이션

- **화면 전환**: 슬라이드 애니메이션 (300ms)
- **카드 로딩**: 스켈레톤 UI
- **보유 상태 토글**: 체크 애니메이션
- **수집률 변경**: 프로그레스 바 애니메이션

## 9. 에러 및 빈 상태

### 9.1 빈 상태

- **빈 카탈로그 목록**: "첫 카탈로그를 만들어보세요" 메시지 + 추가 버튼
- **빈 아이템 목록**: "아이템을 추가해보세요" 메시지 + 추가 버튼

### 9.2 에러 상태

- **네트워크 오류**: "인터넷 연결을 확인해주세요" + 재시도 버튼
- **서버 오류**: "일시적인 오류가 발생했습니다" + 재시도 버튼
- **권한 오류**: "접근 권한이 없습니다" + 홈으로 이동 버튼

### 9.3 로딩 상태

- **전체 화면 로딩**: 중앙 스피너
- **부분 로딩**: 스켈레톤 UI
- **버튼 로딩**: 버튼 내 작은 스피너
