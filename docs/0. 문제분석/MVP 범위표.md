# 카탈로깅 앱 MVP 범위표

본 문서는 초기 단계에 필요한 최소 기능과 비범위(고도화로 이월)를 구분해 명확한 개발 범위를 정의한다.

## 목표와 원칙

- 기간 내 완수 가능한 핵심 문제 해결에 집중
  - 보유 현황 파악, 누락 아이템 식별, 중복 구매 방지에 직접 기여하는 기능만 포함.
- 모바일 1코드베이스, 서버 단일 도메인부터
  - Flutter + Django 단일 배포, 이미지 S3 업로드는 Signed URL 채택.


## 사용자 스토리 기준

- “내 카탈로그를 만들고 아이템을 추가·표시해 보유/미보유를 관리하고, 누락을 한눈에 본다”를 완수할 수 있어야 한다.
- “카테고리별 진행 상황을 간단한 지표로 확인”할 수 있어야 한다.


## 범위 표

| 구분 | 기능 | 상세 범위 | 포함/제외 |
| :-- | :-- | :-- | :-- |
| 인증 | 회원가입/로그인 | OAuth2 기반 SNS 로그인, JWT 발급/갱신, 프로필 최소 필드 | 포함 |
| 카탈로그 | 카탈로그 CRUD | 생성·조회·수정·삭제, 카테고리·정렬 기본값 | 포함 |
| 아이템 | 아이템 CRUD | 이름, 사진, 출시일, 시리즈/태그/기타 정보(선택), 단건/리스트 조회 | 포함 |
| 보유 관리 | 보유/미보유 토글 | 아이템별 토글, 카탈로그별 실시간 수집률 계산·표시 | 포함 |
| 누락 파악 | 미보유 필터 | 시리즈/카탈로그 내 “미보유만 보기” 필터 | 포함 |
| 업로드 | 이미지 업로드 | 모바일 촬영/앨범 업로드 → Signed URL → S3 저장 | 포함 |
| 통계 | 간이 대시보드 | 카탈로그별 수집률(%) 카드 | 포함 |
| 앱 품질 | 오프라인 캐시 | 로컬 DB(SQLite) 캐시, 온라인 복구 시 단순 동기화(서버 우선) | 포함 |
| 보안 | 저장/전송 보안 | HTTPS, JWT 검증, 민감정보 암호화, Signed URL 만료 설정 | 포함 |
| 운영 | 로그/에러 | 공통 에러 포맷, 간단 로깅, 업로드 재시도 큐 | 포함 |
| 검색 | 기본 검색/필터 | 카탈로그/아이템 이름 검색, 카테고리 필터 | 포함(M2) |
| 커뮤니티 | 공개/비공개 | 카탈로그 가시성 설정(비공개 기본, 공개 토글) | 제외(MVP+) |
| 통계 고도화 | 리포트/차트 | 개인화 차트(원형/막대), 카테고리 분포/월별 추세 | 제외(MVP+) |
| 인식 | 이미지 인식/바코드 | 메타데이터 자동 추출·자동 분류 | 제외(MVP+) |
| 실시간 | 알림/스트리밍 | WebSocket/SSE 알림, 푸시 | 제외(MVP+) |
| 인프라 | MSA/컨테이너 | 분리 서비스, 오케스트레이션, 고급 CI/CD | 제외(MVP+) |

## 기술 범위

### 클라이언트(Flutter)

- 상태관리: Provider, 기능 단위 폴더 구조, REST 클라이언트 한 개 엔드포인트 사용.
- 저장소: SQLite 캐싱, 이미지 선택/촬영 및 업로드, 보안 저장소는 후순위(문자열 토큰은 일단 암호화 저장).


### 서버(Django 단일 서비스)

- 앱 모듈: Auth, Catalog, Item, Media(업로드 경로/메타 저장)로 시작.
- API: JWT 기반 인증 미들웨어, 카탈로그/아이템 CRUD, 보유 토글, 수집률 계산 API.


### 스토리지/네트워크

- S3 + CDN, Signed URL 발급, 만료·범위 제한 설정, 전 구간 HTTPS.
- 데이터베이스: PostgreSQL 1 인스턴스(스키마 분리 불요, 단일 서비스), 마이그레이션 관리.


## 마일스톤

- M0 Prototype(2~4주): 카탈로그/아이템 CRUD, 보유 토글, 간이 수집률, 이미지 업로드, 최소 인증.
- M1 MVP(6~8주): 미보유 필터, 오프라인 캐시, 간이 대시보드, 에러/로그, 보안 강화.
- M2 MVP(2주): 기본 검색/필터.

## 추후 고도화 로드맵(MVP+)

- 이미지 인식·바코드 OCR, 개인화 차트 리포트, 커뮤니티 공개/수정 요청, 실시간 알림, 서비스 분리·컨테이너·CI/CD 확장.
- 데이터 품질 파이프라인(야간 배치 집계, 통계 서비스 분리), 권한 세분화/감사 로그.
