# 카탈로깅 앱 사용자 시나리오

본 문서는 “카탈로깅 – 수집가들의 동반 앱”의 핵심 사용자 여정과 상세 시나리오를 마크다운 형식으로 정리한 것이다.

## 가입·온보딩

- SNS 회원가입 후 관심 카테고리(TCG, 레고, 게임, 영화, … 등) 선택.
- 튜토리얼에서 카탈로그 생성, 보유/미보유 토글, 수집률 표시 확인.


## 개인 카탈로그 생성

- “포켓몬 TCG”, “레고 Botanical”, “LP 레코드” 등 카탈로그 생성.
- 카테고리·태그·시리즈 등 메타데이터 설정 및 정렬 기준 지정.


## 아이템 추가·상세 입력

- 이름, 사진, 설명, 태그, 기타 정보(시리즈, 출시일, 희귀도 등) 입력.
- 사진 업로드는 Signed URL로 S3에 직접 저장.


## 보유/미보유 관리·수집률

- 아이템 카드에서 보유/미보유 토글 시 상단 수집률 퍼센티지 즉시 갱신.
- “보유만/미보유만” 필터로 누락 아이템 신속 파악.


## 누락 아이템 파악

- 시리즈별 보기에서 미보유만 리스트업.
- 누락 목록을 위시리스트에 추가하고 우선순위로 구매 계획 수립.

- 온라인 복구 시 동기화, 충돌은 서버 우선 정책으로 병합.

## 검색·필터·정렬(고도화)

- 카탈로그/아이템 이름으로 즉시 검색.
- 카테고리 필터 및 최근 추가 정렬 지원. 그 외 기타 정보 필터/정렬도 고도화 단계에서 지원.


## 멀티 카탈로그·커스텀 컬렉션

- 동일 아이템을 사용자가 테마/시리즈 등으로 분류하여 복수 카탈로그에 포함(e.g. '피카츄' 카드를 '포켓몬 카드'와 '피카츄 굿즈'라는 독립된 테마의 카탈로그들에 포함).
- 카탈로그별 수집률 독립 계산.


## 계정·프로필·보안

- 닉네임·소개·공개 범위 설정, 안전 저장소 기반 토큰 저장 옵션 확인.
- JWT 자동 갱신, 로그아웃/재로그인 지원.


## 장애·예외 흐름

- 업로드 실패 시 재시도 큐 저장 후 자동 재업로드.
- 장애 시 읽기 전용 캐시 제공, 쓰기 요청 보류 또는 안내.


## 관리자·에디터 운영

- 신고/수정 요청 누적 아이템은 관리자 검토 대기열로 이동.
- 승인 시 공용 카탈로그 메타데이터 업데이트로 즉시 반영.

***

### 대표 유저 스토리

- 포켓몬 TCG 카탈로그를 만들고 누락 카드만 모아 예산 계획 수립.
- 레고 시리즈 완성도 퍼센티지 확인, 90% 달성 시 알림 수신.
- 매장에서 바코드 스캔으로 보유 여부 즉시 확인해 중복 구매 방지.
